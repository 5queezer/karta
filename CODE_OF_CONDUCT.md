# Code of Conduct

## Our Values

Karta is an open-source project built on respectful collaboration, technical honesty, and shared curiosity. We want the project to be useful to contributors with different backgrounds, experience levels, and viewpoints. Participation should be safe, constructive, and focused on improving the work.

## Expected Behavior

Please:

- Be respectful, direct, and considerate in discussion and review.
- Assume good faith, but address problems clearly.
- Welcome differing viewpoints and technical disagreement.
- Give and receive feedback without personal attacks.
- Focus on the project, the code, and the evidence.
- Respect boundaries, privacy, and the time of other contributors.

Unacceptable behavior includes harassment, discrimination, intimidation, doxxing, sustained disruption, sexualized language or attention, or other conduct that would reasonably make participation unsafe or hostile.

## Scope

This Code of Conduct applies in all Karta project spaces, including the repository, issues, pull requests, discussions, chats, and any public or private communication reasonably connected to the project. It also applies when someone is representing the project in public.

## Reporting

If you experience or witness a violation, report it to the maintainers through a GitHub issue or by email if a private report is more appropriate. Include relevant context, links, screenshots, or logs where possible.

## Maintainer Response

Maintainers will review reports promptly and handle them as confidentially as practical. They may clarify facts, request additional context, moderate discussions, remove content, restrict participation, or take other action they judge necessary to protect the project and its contributors. Retaliation against reporters is not acceptable.
